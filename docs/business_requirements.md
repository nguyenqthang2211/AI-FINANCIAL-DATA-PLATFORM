# Business Requirements

## 1. Business Context

This project simulates an AI-powered financial transaction analytics platform for a bank or fintech company. The platform is designed to store, process, analyze and monitor financial transactions generated by customers through different channels such as mobile banikng, internet banking, ATM, POS and bank branches.

The main goal is to support transaction analytics, operational monitoring, fraud detection and future AI-powered analysis.

## 2. Target Users

- Data Analyst: analyzes transaction volume, transaction amount, failed rate, top customers and top merchants.
- Data Engineer: designs databases, builds data pipelines, creates data warehouses and processes big data.
- Risk Analyst: monitors suspicious transactions and risk patterns.
- Fraud Analyst: investigates high-risk transactions.
- Product Manager: evaluates payment channel performance and customer behavior.
- Operation Team: monitors transaction status and operational issues.

## 3. Main Business Processes

- Customer registration
- Account opening
- Card issuing
- Financial transaction processing
- Merchant payment
- Transaction fee charging
- Transaction status tracking
- Suspicious transaction monitoring

## 4. Core Data Objects

- Customer
- Account
- Card
- Transaction
- Merchant
- Merchant Category
- Channel
- Transaction Type
- Currency
- Branch

## 5. Business Questions

- What is the total transaction amount by month?
- Which channel has the highest transaction volume?
- Which channel has the highest failed transaction rate?
- Which merchant category generates the most transactions?
- Which customers have the highest transaction amount?
- What time of day has the most transactions?
- How many transactions are successful, failed, pending or reversed?
- Which transactions look suspicious?

## 6. Scope 

In week 1, this project focuses only on designing the OLTP database for the financial transaction system. This includes defining the business requirements, identifying core data objects, designing entities and relationships, creating the SQL schema, inserting sample data and writing validation queries.
