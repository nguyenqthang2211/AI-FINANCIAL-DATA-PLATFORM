# AI Financial Data Platform

End-to-end financial transaction data platform using SQL Server, T-SQL, Spark, Airflow, Kafka, Power BI, Machine Learning, MLOps, and RAG.

## 1. Project Overview

AI Financial Data Platform is a personal portfolio project that simulates how a bank or fintech company manages financial transaction data from operational systems to analytics, fraud detection, machine learning, and AI-powered knowledge retrieval.

The project is designed to practice skills across:

- Data Analysis
- Data Engineering
- Business Intelligence
- Data Science
- MLOps
- AI Engineering with LLM/RAG

## 2. Business Context

The platform stores and analyzes financial transactions generated by customers through multiple channels such as mobile banking, internet banking, ATM, POS, and bank branches.

The main business goals are:

- Analyze transaction volume and transaction amount
- Monitor transaction status and failed transaction rate
- Identify top customers, merchants, channels, and merchant categories
- Detect suspicious transactions and fraud patterns
- Build dashboards for business and risk monitoring
- Build ML models for fraud risk scoring
- Build a RAG assistant for business rules and data documentation

## 8. Local Development

Create a local `.env` file from `.env.example`:

```bash
cp .env.example .env
```

Start SQL Server using Docker:

```bash
docker compose up -d
```

Connect to SQL Server:

```text
Server: localhost,1433
Authentication: SQL Login
User: sa
Password: value from your local .env file
Trust server certificate: Yes
```

Run the OLTP database initialization script:

```text
sql/oltp/00_create_database.sql
```

## 9. Data Privacy

This project uses synthetic data only. No real customer or financial data is used.

Card numbers must be masked and sensitive credentials must not be committed to GitHub.
